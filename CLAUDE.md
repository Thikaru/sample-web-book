# CLAUDE.md

## プロジェクト概要

`sample-web-stack`(Web Stack Lab)は、HTML/CSS/JavaScriptの基礎〜最新技術を
「説明 × 動く見本 × 書き換えられるエディタ」で学ぶ、**育てていく見本集サイト**。
ビルド不要・全ページ静的・日本語UI。`index.html` をブラウザで開くだけで動く。

| ファイル | ページ | 内容 |
|---|---|---|
| `index.html` | トップ | サイト説明・本日の見本/一覧へのボタン・学習進捗 |
| `today.html` | 本日の見本 | 日替わりで見本を1つ表示(タイトル+説明+エディタ) |
| `samples.html` | 見本一覧 | カード表示 + カテゴリ/学習済みフィルタ |
| `sample.html?id=<見本id>` | 見本詳細 | タイトル+説明+ポイント+エディタ |
| `playground.html` | 遊び場 | 自由なエディタ。見本コードの取り込み・自動保存つき |

```
├── css/style.css       … 全ページ共通スタイル(CSS変数でテーマ管理)
├── js/data.js          … ★見本データ。ここに追記すると見本が増える
├── js/details.js       … ★見本ごとの「もっとくわしい解説」(idをキーにしたHTML文字列)
├── js/editor.js        … タブ切替エディタ + ライブプレビューの共通部品
└── js/sample-page.js   … today.html / sample.html の共通描画処理
```

## 見本の追加方法(これが唯一のコンテンツ更新ポイント)

`js/data.js` の `SAMPLES` 配列の末尾にオブジェクトを1つ追加するだけで、
一覧・詳細・本日の見本・遊び場のすべてに自動反映される。

```js
{
  id: "css-grid-layout",            // 一意な英数字ケバブケース(URLに使われる)
  title: "CSS Gridで格子レイアウト",
  category: "CSS",                  // "HTML" | "CSS" | "JS"
  level: "基礎",                    // "基礎" | "応用"
  modern: true,                     // ✨最新技術バッジ(省略可)。Baseline入り時期などを説明に書く
  date: "2026-07-08",               // 追加日 YYYY-MM-DD。当日にすると「本日の見本」に優先表示
  summary: "カード一覧に出す短い説明(1〜2文)。",
  description: "詳細ページに出す技術のしっかりした説明。",
  points: ["学びのポイント1", "ポイント2", "ポイント3"],
  html: `...初期HTMLコード...`,
  css: `...初期CSSコード...`,
  js: `...初期JSコード...`,
},
```

- コードはテンプレートリテラル(バッククォート)で書くため、コード内にバッククォートや `${` を
  含める場合は `\` でエスケープする
- 既存の `id` を重複させない・既存オブジェクトを書き換えない
- 詳しい解説が要るテーマは `js/details.js` の `SAMPLE_DETAILS` に同じ `id` で追加(任意、無ければ非表示)
- `index.html` の学習進捗などは `SAMPLES.length` から自動計算されるため、件数を手動更新する必要はない

## 育てる思想

このサイト自体も学習の題材。見本を追加するだけでなく、学んだ技術を
`css/style.css` のテーマやトップページの演出などサイト自身にも小さく反映していくと良い
(本筋である見本追加を壊さない範囲で)。

## 🗓️ 毎日の自動学習タスク(Claude Cowork スケジュール実行)

このリポジトリは Claude Cowork のスケジュール機能で**毎日自動実行**され、
「新しい見本を1件学んで追加 → GitHubの`main`へ直接push」を行う運用。
毎日実行される際の詳しい手順書は **`docs/daily-learning-instructions.md`** にまとめてあるので、
スケジュール実行時は必ずそちらを読んで従うこと。要点は以下:

1. `js/data.js` の既存 `id`/`title`/`category`/`date` を確認し、まだ無いテーマを1つ選ぶ
   (カテゴリ・レベルの偏りを避ける。事実確認せずBaseline対応時期などを断定しない)
2. README/本ファイル記載の形式で `SAMPLES` 配列末尾に見本オブジェクトを1つ追記(既存分は改変しない)
3. `node --check js/data.js` などで構文確認し、可能ならローカルで表示確認
4. `js/data.js`(+必要なら`js/details.js`)のみをコミットし、`main` へ直接push
   (個人の学習用サイトのため確認不要・PR不要。ただし `git status` で意図しない変更混入がないか確認)
5. 余裕があれば、学んだ技術をサイト本体に小さく反映してもよい(本筋を壊さない範囲で)

やってはいけないこと: 既存見本の上書き・削除、未確認情報の断定、`js/data.js`以外への
無関係な広範囲リファクタリング、動作しないコードの追加。
